# 🔧 Настройка Telegram Client API для отправки от имени пользователя

## 📋 Что нужно для работы

Для отправки сообщений **от имени пользователя** (а не от бота) нужен Telegram Client API.

---

## 🔑 Получение API_ID и API_HASH

### Шаг 1: Зайдите на my.telegram.org

1. Откройте [my.telegram.org](https://my.telegram.org)
2. Войдите используя ваш номер телефона
3. Введите код подтверждения из Telegram

### Шаг 2: Создайте приложение

1. Перейдите в раздел **"API development tools"**
2. Заполните форму:
   - **App title**: Mailing Bot (или любое название)
   - **Short name**: mailing_bot (или любое короткое имя)
   - **Platform**: Desktop
   - **Description**: Bot for sending mailings (опционально)
3. Нажмите **"Create application"**

### Шаг 3: Скопируйте данные

Вы получите:
- **api_id** - число (например: 12345678)
- **api_hash** - строка (например: abc123def456...)

```env

MAIN_ADMIN_ID=120661515

# Новые настройки для Client API
API_ID=ваш_api_id_здесь
API_HASH=ваш_api_hash_здесь
PHONE_NUMBER=+79991234567
```

**Важно:**
- `PHONE_NUMBER` должен быть в международном формате с `+`
- Например: `+79991234567` (для России)
- Или: `+1234567890` (для других стран)

## 🐛 Решение проблем

### Ошибка: "API_ID и API_HASH должны быть установлены"

**Решение:** Проверьте файл `.env`, убедитесь что `API_ID` и `API_HASH` заполнены.

### Ошибка: "Phone number invalid"

**Решение:** Убедитесь что номер в формате `+79991234567` (с плюсом и кодом страны).

### Ошибка авторизации

**Решение:**
1. Убедитесь что номер правильный
2. Проверьте код подтверждения
3. Если включен 2FA - введите пароль

### Ошибка: "FloodWait"

**Решение:** Telegram ограничивает частоту отправки. Бот автоматически ждет нужное время.

## 📊 Как это работает

### Bot API (aiogram):
- ❌ Не может отправлять сообщения пользователям, которые не писали боту
- ✅ Управление через команды

### Client API (Pyrogram):
- ✅ Может отправлять сообщения **любым** пользователям
- ✅ От имени вашего аккаунта (не от бота)
- ✅ Не требует чтобы получатель писал боту

### Гибридный подход:
- **Управление** → через Bot API (aiogram)
- **Рассылки** → через Client API (Pyrogram)
