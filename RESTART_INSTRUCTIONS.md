# 🔄 Инструкция по перезапуску бота после изменений

## ⚠️ Важно: После изменения кода нужно перезапустить бота!

### Проблема:
Если вы изменили файлы, но не перезапустили бота - изменения **НЕ применятся**!

---

## ✅ Правильный процесс:

### 1. Остановите бота

В терминале, где запущен бот:
```
Нажмите: Ctrl + C
```

Должно появиться:
```
^C
Бот остановлен пользователем
```

### 2. Убедитесь, что бот остановлен

Проверьте, что процесс завершен:
```bash
ps aux | grep "python3 main.py"
```

Если видите процесс - убейте его:
```bash
pkill -f "python3 main.py"
```

### 3. Очистите кэш Python (опционально, но рекомендуется)

```bash
cd /Users/aspbr/Documents/botdubaiterra4

# Удалите кэш Python
find . -type d -name __pycache__ -exec rm -r {} + 2>/dev/null
find . -type f -name "*.pyc" -delete 2>/dev/null
```

### 4. Запустите бота заново

```bash
python3 main.py
```

---

## 🔍 Проверка, что изменения применились:

### 1. Проверьте логи при запуске

Должно быть:
```
2025-12-21 ... - INFO - Запуск бота...
2025-12-21 ... - INFO - Инициализация базы данных...
2025-12-21 ... - INFO - База данных инициализирована
2025-12-21 ... - INFO - Бот запущен. Администратор: 120661515
```

### 2. Проверьте в Telegram

1. Отправьте `/start` боту
2. Проверьте кнопки:
   - "⚙️ Настройки" - должна показать настройки
   - "ℹ️ Помощь" - должна показать подробную справку
   - "📊 Мои рассылки" - должна показать список или сообщение
   - "📧 Новая рассылка" - должна показать шаблоны

---

## 🐛 Если кнопки все еще не работают:

### Проверка 1: Убедитесь, что файлы сохранены

```bash
# Проверьте дату изменения файлов
ls -la handlers/user.py handlers/admin.py
```

Должны быть свежие (только что измененные).

### Проверка 2: Проверьте синтаксис

```bash
python3 -m py_compile handlers/user.py handlers/admin.py
```

Если есть ошибки - исправьте их.

### Проверка 3: Проверьте импорты

```bash
python3 -c "from handlers import admin, user, mailing; print('OK')"
```

Должно вывести: `OK`

### Проверка 4: Проверьте, что вы администратор

В Telegram отправьте:
```
/start
```

Проверьте, что в приветствии написано:
```
🔑 Вы являетесь администратором...
```

Если нет - проверьте `.env` файл:
```bash
cat .env | grep MAIN_ADMIN_ID
```

Должно быть: `MAIN_ADMIN_ID=120661515`

## 🔄 Полный процесс перезапуска (скрипт):

Создайте файл `restart.sh`:

```bash
#!/bin/bash
cd /Users/aspbr/Documents/botdubaiterra4

echo "🛑 Останавливаем бота..."
pkill -f "python3 main.py" 2>/dev/null
sleep 2

echo "🧹 Очищаем кэш..."
find . -type d -name __pycache__ -exec rm -r {} + 2>/dev/null
find . -type f -name "*.pyc" -delete 2>/dev/null

echo "🚀 Запускаем бота..."
python3 main.py
```

Сделайте исполняемым:
```bash
chmod +x restart.sh
```

Используйте:
```bash
./restart.sh
```

## ⚡ Быстрый перезапуск:

```bash
# В терминале с ботом
Ctrl+C  # Остановить

# Подождите 2 секунды, затем:
python3 main.py  # Запустить
```

---

## 💡 Совет:

**Всегда перезапускайте бота после изменения кода!**

Python загружает модули один раз при старте. Изменения в файлах не применяются автоматически.

1. **Изменили код** → Сохранили файлы
2. **Остановили бота** → Ctrl+C
3. **Запустили бота** → `python3 main.py`
4. **Проверили** → Кнопки работают
