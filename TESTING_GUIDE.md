# 🧪 Руководство по тестированию бота

## 🔍 Как проверить работу после исправлений

### 1. Перезапуск бота

```bash
cd /Users/aspbr/Documents/botdubaiterra4

# Остановите бота (Ctrl+C)
# Запустите заново:
python3 main.py
```

### 2. Тест выбора шаблона

**Шаги:**
1. Отправьте `/start` боту
2. Нажмите "📧 Новая рассылка"
3. Выберите шаблон (нажмите на кнопку)
4. **Ожидаемый результат:** Должно появиться сообщение с запросом ввода получателей

**Что проверить в логах:**
```
Шаг 1: template_id X сохранен в state
Шаг 2: Состояние установлено в waiting_for_recipients
Шаг 3: Сообщение с инструкциями отправлено
Шаг 4: Текущее состояние: MailingStates:waiting_for_recipients
```

### 3. Тест ввода получателей

**Шаги:**
1. После выбора шаблона введите получателя
2. Например: `@ваш_username` или `120661515`
3. **Ожидаемый результат:** Должно появиться превью рассылки

**Что проверить в логах:**
```
Получено сообщение от X в состоянии waiting_for_recipients
Пользователь X ввел список получателей: 1 получателей
```

### 4. Если не работает

**Проверьте логи:**
- Есть ли ошибки?
- Устанавливается ли состояние?
- Отправляется ли сообщение?

**Проверьте состояние:**
- Отправьте `/start` заново
- Попробуйте еще раз

## 🐛 Отладка

### Включение подробных логов

В `config.py` измените:
```python
LOG_LEVEL = "DEBUG"  # вместо "INFO"
```

### Проверка состояния FSM

Добавьте временный handler:
```python
@router.message(Command("state"))
async def check_state(message: Message, state: FSMContext):
    current = await state.get_state()
    data = await state.get_data()
    await message.answer(f"Состояние: {current}\nДанные: {data}")
```
